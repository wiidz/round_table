# v0.2.0 — First public preview (Meeting Engine + Discord + Web)

> GitHub Release 正文可直接复制下方「Release notes」区块（从 `## 概述` 到文末）。

## 概述 / Overview

**中文：** RoundTable 首个对外可试用版本——多智能体**会议引擎**，支持在 **Discord** 跑完整会议流程，并用 **Web 工作台**浏览与管理会议产出。本版本对齐路线图中的 **v0.2**（Discord 切片 + Web 工作台增量）。

**English:** First public preview of RoundTable — a **multi-agent meeting engine** with full meetings on **Discord** and a **Web workbench** to browse and manage meeting outputs. Aligns with **v0.2** on the roadmap (Discord slice + Web workbench increment).

---

## 本版本能做什么 / What's included

- **Meeting Engine** — Pre-meeting (R0), multi-round debate/deliberation, free dialogue, Moderator summaries, consensus/synthesis, Principal confirmation (approve/reject with extra rounds), workspace artifacts, token usage, abort & startup reconcile
- **Discord Transport** — Principal bind, natural language / `!rt` launch, brief wizard, preset menu, brief template (`-template`), confirmation gate, runtime interventions, Principal Q&A during free dialogue, executive recap, Chinese i18n
- **Web workbench** — Meeting list & detail (overview, three-column layout), documents with TOC, flow timeline (including rejection steps), round-table live & replay, brief template library, Principal persona editing, meeting archive download & delete, site-wide zh/en i18n
- **WebChat Transport** — WS `/api/chat/ws` for browser-based meetings (no Principal bind required)
- **Demo without API keys** — `make seed-demo` then browse `mtg-demo-001` locally
- **Docker deploy** — `make docker-up` for Web + API + Discord Supervisor on Linux servers
- **Docs & OSS** — Apache 2.0, CI, CONTRIBUTING, bilingual README ([English](../../README.md) · [中文](../../README.zh-CN.md))

## 快速开始 / Quick start

**Browse the UI only (no API key):**

```bash
git clone https://github.com/wiidz/round_table.git
cd round_table
make seed-demo
make server-dev   # terminal 1
make web-dev      # terminal 2 → http://localhost:5173
```

Open **Meetings** → `mtg-demo-001` for overview, documents, flow, and replay. Explore **Brief templates** (`/brief-templates`) and **Principals** (`/principals`) from the sidebar.

**Docker (server):** see [deploy/README.md](../../deploy/README.md)

**Full guide:** [docs/getting-started.md](../getting-started.md)

## 当前限制 / Current limitations

| Area | Status |
|------|--------|
| **LLM** | [DeepSeek](https://platform.deepseek.com/) only (`DEEPSEEK_API_KEY`) |
| **Transport** | **Discord** for full live meetings; **Web** for browsing/replay + WebChat transport |
| **Web gaps** | No full meeting launch wizard parity with Discord; Confirmation ItemNotes editing not yet in Web UI |
| **Planned** | More models & transports (Slack, enterprise IM, …) — see [roadmap](../roadmap.md) |

CLI `apps/server/cmd/meet` is for **developer debugging**, not the primary end-user path.

---

## 核心角色 / Core roles

| Role | One line |
|------|----------|
| **Principal** | Human decision-maker; starts meetings and holds final acceptance |
| **Moderator** | Facilitator agent; runs the room, does not own domain expertise |
| **Participant** | Expert agents; speak only when invited |
| **Meeting** | Structured discussion from pre-meeting through closure and artifacts |

## 自开源准备包以来的主要变更 / Highlights since open-source prep

- Bilingual README with quick-start upfront; core concepts & current-support sections
- Meeting detail UX: localized file labels, confirmation rejection in flow, collapsible sidebars
- Brief template Web pages + Discord `-template` apply; `clone-brief` from historical meetings
- Principal Persona editing (`USER.md` structured fields) on Web
- Meeting workspace archive download & delete (API + Web UI)
- Site-wide Web i18n (`ROUND_TABLE_LOCALE` zh/en)
- Round-table live view (seats, highlights, transcript strip/drawer) and meeting replay
- CI: Go + Web (Vitest + production build)
- Demo seed script (`make seed-demo`)
